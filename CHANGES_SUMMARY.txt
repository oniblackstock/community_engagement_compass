================================================================================
PROBLEM STATEMENT
================================================================================

The Knowledge Assistant was displaying raw HTML tags (<p>, <strong>, <ul>, etc.)
instead of rendering them as formatted text. Additionally, the system lacked
proper XSS (Cross-Site Scripting) protection.

Example before fix:
  User sees: "<p>Hello <strong>world</strong></p>"
  Expected:  "Hello world" (with formatting)

================================================================================
SOLUTION IMPLEMENTED
================================================================================

Integrated DOMPurify library for safe HTML sanitization and updated the
formatMessage() function in the frontend to properly render HTML while
preventing XSS attacks.

Key changes:
  1. Added DOMPurify CDN script to chat template
  2. Updated formatMessage() with sanitization logic
  3. Configured whitelisted HTML tags and attributes
  4. Added fallback for CDN failures

ALLOWED HTML ELEMENTS
================================================================================

ALLOWED TAGS:
  ✓ Structural: <p>, <br>
  ✓ Headings: <h1>, <h2>, <h3>, <h4>, <h5>, <h6>
  ✓ Lists: <ul>, <ol>, <li>
  ✓ Text Formatting: <strong>, <em>
  ✓ Code: <code>, <pre>
  ✓ Quotes: <blockquote>
  ✓ Links: <a>

ALLOWED ATTRIBUTES:
  ✓ href (for links)
  ✓ target (for link targets)
  ✓ rel (for link relationships)

BLOCKED ELEMENTS:
  ✗ <script> - Prevents JavaScript execution
  ✗ <iframe> - Prevents frame injection
  ✗ <img> - Prevents image injection
  ✗ Event handlers - onclick, onerror, onload, etc.
  ✗ javascript: protocol - Blocks JavaScript URLs
